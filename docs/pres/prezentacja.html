O projekcie

KONTEKST

Cel pracy

  • Zaprojektowanie i realizacja platformy SaaS do zarzadzania usluga VPN
  • Centralne zarzadzanie flota serwerow VPN w chmurze AWS

  • PKI — certyfikaty x509 generowane przez rcgen (Rust)
03 / 16

Technologie VPN — przeglad

PROTOKOLY

WireGuard

  • Nowoczesny protokol — ~4000 LOC w jadrze Linuxa
  • Warstwa 3 (IP), interfejs TUN
  • Szyfrowanie: ChaCha20-Poly1305
  • Wymiana kluczy: Curve25519 (ECDH)
  • Auth: para kluczy publicznych
  • Najszybszy, najnizszy narzut

Glowny protokol — domyslnie wlaczony

OpenVPN

  • Dojrzaly protokol, dziala w user space
  • Warstwa 3, interfejs TUN (tun1)
  • Szyfrowanie: AES-256-GCM
  • Auth TLS: RSA 2048 (self-signed)
  • Auth klienta: username/password
  • Szeroka kompatybilnosc platform

Drugi protokol — fallback

IKEv2 / IPsec

  • Protokol IPsec z IKEv2 key exchange
  • Warstwa kernela, strongSwan (serwer)
  • Szyfrowanie: AES-256-GCM
  • Wymiana kluczy: ECP256 (ECDH)
  • Wbudowany w OS (macOS, Windows)
  • Szybki reconnect — MOBIKE

Trzeci protokol — czesciowy

WireGuard — implementacja

PROTOKOL VPN

Serwer — vpn/wireguard.rs

  • WireGuardBackend::new(port) — config dir: /etc/wireguard
  • Klucz prywatny: Command::new("wg").arg("genkey") → server.key
  • Klucz publiczny: Command::new("wg").arg("pubkey") (stdin: privkey)
  • Interfejs: ip link add wg0 type wireguard lub wireguard-go fallback
  • Konfiguracja: wg setconf wg0 wg0.conf + ip addr add 10.8.0.1/24
  • Start: ip link set up dev wg0

apply_peers() — sync co 2s

  • Control plane zwraca liste PeerSpec z public_key + allowed_ips
  • Buduje nowy wg0.conf z sekcjami [Peer]
  • Zapis do wg0.conf.new (atomowy swap)
  • Linux/macOS: wg syncconf wg0 wg0.conf.new — live update bez przerwy
  • Fallback: pelny wg setconf jesli syncconf zawiedzie
  • Windows: podmiana conf + restart serwisu

Serwer — start() (wireguard.rs:151)

fn start(&self) -> Result<()> {
  self.ensure_keys_and_config()?;
  // wg genkey → /etc/wireguard/server.key
  // wg0.conf: [Interface] PrivateKey + ListenPort

  let nm = get_network_manager();
  nm.configure_wireguard(
    "wg0", "10.8.0.1/24", self.listen_port,
    &self.conf_path()
  )?; // ip link add + wg setconf
  nm.bring_up_interface("wg0")?;
  // ip link set up dev wg0
}

OpenVPN — implementacja

PROTOKOL VPN

Serwer — vpn/openvpn.rs

  • Self-signed cert: openssl req -x509 -newkey rsa:2048 -sha256 -days 3650
  • Serwer nasuchuje na porcie 1194/UDP (env OPENVPN_PORT)
  • Auth klienta: auth-user-pass-verify verify.sh via-env
  • Skrypt verify.sh: grep -F -x "$username $password" secrets.txt
  • Management interface na 127.0.0.1:7505 do monitoringu
  • Status: parsowanie /var/run/openvpn-status.log (wersja 3)

apply_peers() — aktualizacja credentials

  • Iteruje po PeerSpec — filtruje peery z username + password
  • Zapisuje do /etc/openvpn/secrets.txt: "user pass\n" per linia
  • OpenVPN czyta plik przy kazdej probie auth — nie wymaga reloadu
  • Metryki: CLIENT_LIST,CN,Addr,BytesRx,BytesTx z status.log

Konfiguracja serwera: proto udp, dev tun1, server 10.9.0.0/24, topology subnet, verify-client-cert none, username-as-common-name

Serwer — konfiguracja (openvpn.rs:52)

IKEv2 / IPsec — implementacja

PROTOKOL VPN

Serwer — ikev2.rs + vpn/ipsec.rs

  • setup_ikev2() kopiuje certyfikaty PKI do /etc/ipsec.d/
  • CA → /etc/ipsec.d/cacerts/ca.pem
  • Server cert → /etc/ipsec.d/certs/server.pem
  • Klucz prywatny → /etc/ipsec.d/private/server.key
  • Generuje /etc/ipsec.conf i /etc/ipsec.secrets
  • Start: ipsec start --nofork (spawn jako child process)
  • Status: swanctl -l → zlicza ESTABLISHED / INSTALLED

Generowanie certyfikatow — pki.rs

use rcgen::{Certificate, CertificateParams,
            IsCa, BasicConstraints, SanType};
use sha2::{Digest, Sha256};

// 1. CA certificate (self-signed)
let ca_params = CertificateParams::default();
ca_params.is_ca = IsCa::Ca(Unconstrained);
ca_params.distinguished_name
  .push(DnType::CommonName, "vpnvpn-ca");
let ca = Certificate::from_params(ca_params)?;

// 2. Server cert (signed by CA)
let srv = CertificateParams::new(vec!["vpnvpn-server"]);
srv.subject_alt_names.push(SanType::IpAddress(ip));
let pem = srv.serialize_pem_with_signer(&ca)?;

// 3. Fingerprint (SHA-256, hex format)
let digest = Sha256::digest(&der);
// "AA:BB:CC:..." → rejestrowany w control plane

let protos_env = env::var("VPN_PROTOCOLS").unwrap_or("wireguard,openvpn,ikev2");
